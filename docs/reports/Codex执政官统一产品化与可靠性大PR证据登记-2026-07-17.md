## 四、证据缺口和不确定性

- OpenAI 官方页面在本机 static/dynamic extraction 都出现 HTTP 403/内容不足，因此关于当前 Codex Desktop 内部刷新时序只采用官方搜索摘要、公开 issue 和本地现场事实的交集，不声称有内部 API 保证。
- GitHub issue 反映的是公开用户报告和当前状态，不是 OpenAI 的修复时间表或稳定产品合同。
- Windows Restart Manager 的进程识别需要资源登记，不能保证覆盖所有 Codex 会话引用；因此报告坚持“记录 blocker，不强杀”。
- provider CLI 文档描述官方语义，但不同发行版、版本、账号域和模型可能有差异；最终 readiness 仍必须在本机当前 tuple 上运行真实 canary。
- 学术搜索只得到元数据或摘要线索时，不把它们作为某个具体实现的唯一依据。
