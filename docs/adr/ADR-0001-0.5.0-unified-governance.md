# ADR-0001：0.5.0-alpha 统一治理控制面

日期：2026-07-18
状态：已接受

## 决定

Codex Praetor 保留现有 logical task、attempt、event、job 和 active receipt，增加明确的 completion definition、write set、budget、stop-loss、selection、outcome、progress 和 release state 字段。状态按四层记录：

1. 执行层：`draft`、`queued`、`running`、`cancel_requested`、`process_exited`。
2. 证据层：报告、工件、测试和缺证据状态。
3. 治理层：由 Codex 验收的 `awaiting_supervisor`、`accepted`、`rejected`、`retryable`、`blocked`、`needs_decision`。
4. 发布层：`built`、`published`、`staged`、`awaiting_evidence`、`active`、`delivery_blocked`、`delivered`。

任何进程退出、completion.json、GitHub Release 或 active receipt 都不能单独把逻辑任务或产品标记为完成。只有 Codex 验收事件和发布控制器的完整门禁才能推进终态。

## 约束

- 账本事件追加写入；旧 plan/job 只做兼容读取，不把历史成功重新解释为已验收。
- selection 只记录透明规则的候选、理由、预算和升级条件，不自动学习并静默改路由。
- readiness 只回答“当前是否能安全执行指定 tuple”；selection 和 outcome 分开记录。
- health 读取当前 readiness、CLI hash、过期时间、maintenance task action/triggers/enabled 和 receipt，不使用历史 receipt 快照冒充动态真值。
- 默认只读生成 runtime inventory，禁止删除 dirty/unmerged、active 或审计保留项。

## 明确排除

本版本不引入数据库、Redis、Temporal、A2A server、通用多 agent 平台、worker 群聊或第二个总控 agent。Codex 继续负责规划、监督和验收，外部 CLI worker 仅执行受限 attempt。

## 验收依据

- `config/task-governance.schema.json` 必须能表达四层状态及工件/预算/止损字段。
- 进程退出但没有 supervisor verdict 时，依赖任务不得 ready。
- readiness 过期、CLI hash 漂移、generation/runtime tuple 不匹配时，dispatch 和 health 都 fail closed。
- maintenance task 缺失、禁用、action/arguments/triggers 漂移时，stable health 必须 degraded 或 blocked。
