# Codex Praetor 0.9.1-alpha

本版把“真实验证”从临时提示词变成了可复跑的实验合同。它仍不会假装三家 provider 都已经通过真实任务；真实结果必须由 worker、diff、测试和 Codex 验收共同产生。

## 主要改动

- 新增版本化真实任务库，覆盖只读诊断、受限改码、固定测试和失败恢复四类工作。
- 每个任务明确允许/禁止路径、候选 provider、验收、确定性检查、预算、失败注入和不接触认证材料的边界。
- 新增项目内 evaluation plan 准备器：只建立待执行账本，不会批量派工，也不会把准备动作误记为能力证据。
- 新增只读 MCP 工具 `codex_praetor_evaluation_suite`，让 Codex 可读取实验合同和边界。
- 将实验合同回归加入候选与发布流水线。

## 用户影响

后续真实 worker 验收不再依赖很长、很脆弱的提示词，而会使用短业务目标配合外层任务合同。每次失败也能被保留为可解释证据，主项目代码不会被实验 worktree 污染。

## 发布与验收

合并后 `Release On Main` 必须从精确 merge commit 自动发布新的不可变 zip，并完成远端下载验真和稳定安装自动更新。Desktop host 刷新仍按四个 PR 的统一最终验收安排执行。
