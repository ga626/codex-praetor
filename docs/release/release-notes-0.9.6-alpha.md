# Codex Praetor 0.9.6-alpha

## 本次修复

- 修复真实评测派工链路：以前“需要运行固定测试”的任务在计划台账中会丢失任务类别，随后被误当成纯阅读审计，worker 因此没有命令执行工具。
- 现在这类任务使用独立的 `test_execution` 合同：只允许读取、搜索和运行已声明的固定检查，不允许编辑或写入源码。
- 计划台账会完整保存任务类别、允许/禁止路径、必跑检查、预算、故障注入和敏感度；缺任一项或类别与模式冲突时，系统会在启动 worker 前拒绝派工，而不会猜测或放宽权限。
- 修复 Windows PowerShell 环境差异：发布、安装、派工和验收所需的文件完整性校验会在 `Get-FileHash` 不存在时自动改用 .NET SHA-256；固定评测已强制走过这条兼容路径。

## 用户可感知到的变化

真实评测中的“跑固定回归测试”任务能够获得刚好够用的命令执行能力；普通只读审计仍然不能执行 shell，代码修改任务仍然必须使用隔离 worktree 和编辑合同。三类任务不再互相混淆。

## 发布与本机验收

这是新的不可变版本，不覆盖已有 tag 或 Release。合并后，`Release On Main` 将从 merge SHA 自动发布 `v0.9.6-alpha`，再由同一公开包完成远端验真与 stable marketplace 自动更新。根据本轮四个 PR 的总体验收约定，本机 host 刷新、`runtime_info`、最终 canary 和真实 worker 体验会在第 4 个 PR 发布后集中进行。
