# Codex Praetor 0.13.0-alpha

## 本次交付

- 正常派工不再只参考一段历史记录。Qoder 或 CodeBuddy 必须在同一任务族、同一 CLI、同一模型、同一权限和同一运行时合同下，留下至少三份新鲜且由 Codex accepted 的证据，才可接收正常任务。
- 证据现在保存在用户级账本中，而不是留在某个临时工作树；新对话和新的隔离工作树可以读取同一份已验收事实。
- 证据不足、过期、格式损坏、模型/权限/CLI 不一致时，系统明确阻止正常派工；只能由 Codex 发起标明任务族、范围可控的 canary，不能把验证伪装成正式任务。
- 历史 plan 仍可供审计，但不再被当成新对话中的派工授权。

## 用户可见边界

- 这不会自动把任务派给任何 provider，也不会自动合并或发布。Codex 仍负责选择工作包、审查结果和执行最终验收。
- Qoder 与 CodeBuddy 仍是当前仅有的公开 worker；本版不会读取账号材料、不会自动登录，也不会修改 provider 设置。
- 旧的成功记录可能不足以立刻放行新任务，这是刻意的安全收紧：它们缺少完整 worker 身份，必须以新的受控 canary 重新建立证据。
