# Codex Praetor 0.10.0-alpha

发布日期：2026-07-23

## 本次改动

- 新增公开能力清单：每项用户能力现在声明入口、包内依赖、最终包场景和验证级别；发布检查不再只确认工具或文件存在。
- 新增 `codex_praetor_prepare_evaluation`：已安装插件可把内置真实评测清单准备为带任务类别、权限、范围、检查和预算的项目内计划；准备计划不会派工，也不是能力证据。
- 最终 zip 会验证公开能力清单、评测读取与评测准备；清单或关键数据被删除时，故障注入必须失败。
- 精简全局与项目规则：固定 Codex 对范围变化、PR 就绪、发布与收口的处理机制，不要求用户使用模板表达需求。

## 用户影响

安装后，Codex 可以直接准备可验收的真实任务评测计划，不再只能查看清单。后续方向、问题和范围仍可在任意时点自然补充；会影响用户承诺或发布边界的变化会在同一 PR 内重新对齐并验证。

## 验收与发布

`Release On Main` 会从合并 SHA 构建唯一 zip，执行公开能力矩阵、故障注入、远端下载复验和自动本机激活。安装完成后需刷新一次 Codex host，随后在新任务中核对 `runtime_info` 再运行 canary/真实 worker。
